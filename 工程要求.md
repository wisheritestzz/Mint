# 一、工程前准备
1、在实际执行前，你先充分理解需求，对不理解不确认的要先找我核实清楚
2、理解完成后，你要自己先生成一份任务跟进文件，确保你后续能持续严谨按文件内容迭代开发，避免上下文过长遗漏信息
# 二、工程中的要求
1、过程中遇到执行不下去，或者需要我这边授权、配置等的工作，要找我确认
2、确保充分实现需求文档中要求的内容，禁止擅自更改缩减需求
3、重视样式、交互审美，功能样式、交互一定要有设计感，理解需求后可以按需研究市面上优秀的组件库/网页，或者github里好用的skill
4、重视代码的规范性、合理性、美观度，追求极致的代码效率，保障架构合理性的情况下，要以尽可能少的代码实现目标需求。可以按需下载参考github里好用的skill
5、开发完成后要进行严谨细致的测试回归，制定严谨的测试用例，多轮回归验证，确保没问题后再让我验收
